## Aula City
Anotações da aula do city - 11/03/2022 - <strong>Camadas do Desenvolvimento de Sistemas</strong>

### Levantamento
- O que deve ser feito?  
- Saiba o que o ciente quer.
- Entenda a necessidade do cliente.
- Entenda o negócio por trás do cliente.
- Regra de negócio.

### Análise 
- Processamento dos dados e informações exibidas pelo cliente.
- Diagramas: Casos de Uso, Diagrama de Sequência.

### Modelagem
- Modalagem do banco de dados seguindo a análise.
- Diagramas: Conceitual, Lógico e Físico.
- Opcional: Modelar entidades do sistema.
- Provisionar o diagrama de classes seguindo o bando de dados.

### Protótipo
- Design do sistema.
- Como precisa-se parecer?
- Provisionar as telas do sistemas.
- Ferramentas: Figma.

### Desenvolvimento
- Plataforma - Ex: Java, NodeJS
- Use a plataforma de acordo com a necessidade.

### Teste de Qualidade
- Teste o programa em desenvolvimento.

### Homologar / Deploy / CI CD
- Saber a plataforma.
- Colocar no ar o site (deploy).

### Paradigmas de Programação 
- POO - Abstração de um objeto e reaproveitamento de código. Pilares da POO: Abstração, Encapsulamento, Herança e Polimorfismo. 

### Termos citados
- README: Mostra configurações e uma "visão geral" do sistema apresentado.
- Protifólio profissional próprio (github).
- Load Balance.
- VPN: Virtual Private Network. 
- Proxy.
- IP dinâmico e fixo.
- API: Application Program Interface. Aplicação que geralmente utiliza um banco de dados e apenas disponibiliza os dados para outras aplicações.
Ex - API CEP: https://viacep.com.br/01001000/json || https://viacep.com.br/<code>&lt;coloque seu cep aqui&gt;</code>/json
Ex - Link de Mapa do Google, ou seja, link de incorporação do HTML para utilizar na sua página.
